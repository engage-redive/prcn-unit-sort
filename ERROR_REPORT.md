# プロジェクトエラー分析レポート

## ステップ1：プロジェクト全体のエラー一覧

TypeScriptの型チェック（`tsc`）およびESLintの実行結果に基づき、以下のエラーを確認しました。

1. **ESLint設定エラー**:
   - `eslint.config.js` で `typescript-eslint` パッケージが見つからないため、リンターが動作を停止しています。
2. **型定義不整合（数千件）**:
   - `src/data/pokedex.ts`, `moves.ts`, `abilities.ts` 内で、リテラル文字列（例: `"fire"`, `"water"`）を `PokemonType` 型に代入しようとしています。
   - `PokemonType` が `const enum` として定義されているため、TypeScriptが文字列との互換性を認めていません。
3. **App.tsx におけるロジック型エラー**:
   - タプル型 `[PokemonType, (PokemonType | undefined)?]` を配列型 `PokemonType[]` に代入しようとしています（`undefined` が含まれる可能性があるため）。
   - `StatCalculationSnapshot` の `nature` プロパティに `number` 型を代入しようとしています（期待される型は `NatureModifier` 即ち `0.9 | 1.0 | 1.1`）。
4. **ユーティリティ関数の型エラー**:
   - `src/utils/uiHelpers.ts` で、型ガードの結果 `never` 型になった変数に対して `.toString()` を呼び出そうとしています。
5. **未使用の変数・インポート（多数）**:
   - `src/App.tsx`, `src/utils/moveEffects.ts` 等で、宣言されているが使用されていない変数が多数存在します。

## ステップ2：原因分類と優先順位レポート

| 優先順位 | エラー分類 | 主な原因 | 影響範囲 | 解決策（推奨） |
| :--- | :--- | :--- | :--- | :--- |
| **最優先 (P0)** | **環境・ツール設定** | `typescript-eslint` パッケージの不足 | ESLintが機能せず、開発中のコード品質チェックが不能。 | パッケージのインストール、または `eslint.config.js` の記述を既存パッケージに合わせる。 |
| **高 (P1)** | **データ定義 (型不整合)** | `PokemonType` 等を `const enum` で定義している | プロジェクトのビルドが通りません（約4,000件のエラー）。 | `PokemonType` をリテラル文字列のユニオン型（`type PokemonType = 'fire' \| ...`）に変更する。 |
| **中 (P2)** | **ロジック・型変換** | 厳密な型チェック（Strict Mode）による不一致 | `App.tsx` 等の主要コンポーネントで予期せぬ動作やビルドエラーが発生。 | `NatureModifier` への適切な型キャスト、またはタプルから `undefined` を除去する処理の追加。 |
| **低 (P3)** | **コードクリーンアップ** | 未使用のコードが残っている | ビルド時の警告およびコードの可読性低下。 | 不要なインポートの削除、または未使用変数への `_` 接頭辞の付与。 |

### 総評
現在、プロジェクトは「型定義の不整合」によってビルド不能な状態にあります。特に **PokemonType の定義見直し (P1)** を行うだけで、全エラーの 90% 以上が解消される見込みです。また、開発環境を整えるために **ESLint の依存関係修正 (P0)** も急務です。
